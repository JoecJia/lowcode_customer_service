# Skill: 知识检索 (Knowledge Retrieval)

## 描述
这是智能客服系统的核心检索能力。当用户提出关于产品功能、业务逻辑、账号操作或常见问题时，总调度智能体应调用此技能来获取精准的知识片段。

## 输入参数
- `query` (string): 用户咨询的核心关键词或问题摘要。

## 执行逻辑
1. **查阅索引**：读取 `context/index.md`，搜索与 `query` 匹配的关键词或类别。
2. **定位文件**：根据索引找到具体的 `.md` 文件路径。
3. **提取片段**：读取目标文件，仅截取与问题直接相关的段落（Header 及其下属内容）。
4. **提取图片信息**：
   - 从命中片段中提取所有 Markdown 图片引用 `![alt](path)`。
   - 输出图片的 `alt` 与 `path`（保留原始相对路径或绝对路径）。
   - 若同一图片在片段中重复出现，仅保留一条。
5. **返回结果**：返回“命中文本 + 命中图片（可选）”，不带无关信息。

## 输出格式
按以下结构返回，方便总调度智能体与业务咨询类技能拼装最终答复：

### 命中文本
（与问题直接相关的段落内容，保留必要的小节标题）

### 命中图片（可选）
- `alt`: 图片说明文字（可为空）
- `path`: 图片路径（例如：`../assets/form_usage_guide/image156.png`）
- `source`: 命中来源文件（例如：`context/product_docs/form_usage_guide.md`）

## 使用建议
- 若命中片段中包含图片，且图片能显著提升用户理解（例如配置步骤截图、对比示意、效果图），总调度智能体在最终回复中应一并输出图片信息。
- 若用户侧无法直接查看图片，可优先输出图片的说明文字与路径，告知用户到对应文档位置查看。

## 优势
- **节省上下文**：避免将整个知识库塞入对话。
- **提高准确性**：强制通过索引定位，减少模型“幻觉”。

## 使用示例
- 用户问：“怎么导出数据？”
- Skill 动作：匹配索引中的“数据导出”，定位到 `context/faq/general_faq.md`，提取导出相关的 Q&A 段落。
