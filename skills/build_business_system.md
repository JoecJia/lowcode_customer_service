# Skill: 搭建具体业务系统指南 (Business System Building Guide)

## 描述
用于引导用户在低代码平台上一步步搭建完整的、具有特定行业特征的业务系统（如：学工管理系统、实习系统等）。

## 输入参数
- `system_type` (string): 目标系统类型（如“学工系统”）。
- `detail_level` (string): 指南的详细程度。

## 执行逻辑
0. **需求预审**：评估用户提供的信息是否明确。若发现信息不全，可调用 `knowledge_retrieval` (知识检索) 查询相关 Context，或调用 `clarifying_questions` (信息补充与反问) 向用户询问必要信息。
1. **步骤拆解与确认**：
   - **第一步：需求拆解与角色定义**：
     - 根据用户描述拆解具体需求点。
     - 明确系统包含的角色（如：学生、辅导员、管理员）。
     - 定义各角色功能及其操作环境（前台/后台）。
     - 划分各角色权限。
     - 复杂流程需输出 Markdown 格式流程图。
     - **确认逻辑**：将上述内容列出给用户确认，可能需多次磨合，完全确认后方可进入第二步。
   - **第二步：数据建模与 ER 设计**：
     - 确定需创建的表及字段。
     - 建立表间关联及数据流动关系。
     - 使用 Markdown 绘制 ER 图并结合描述讲解设计思路，需一一对应第一步的需求点。
     - **迭代处理**：用户调整建模可能会影响第一步需求，需及时反问或同步更新。
     - **确认逻辑**：描述结果给用户确认，完全确认后进入第三步。
   - **第三步：详细配置指南输出**：
     - **功能确认与检索**：调用 `knowledge_retrieval` (知识检索) 技能，确认实现需求所需的具体产品功能（如：数据联动、公式计算、业务流、连通器等）。
     - **建表详细描述**：
       - 明确每个表中的字段具体使用表单中的哪些字段类型创建（如：单行输入、数字、下拉框、联系人、部门、子表单等）。
       - 字段类型的选择需参考 `knowledge_retrieval` 检索到的产品规格。
     - **详细配置描述**：给出每个功能的具体配置方式。
       - *示例*：A 表“班主任”字段配置“数据联动”，联动 B 表“班主任”字段，联动条件为（姓名 等于 姓名）且（班级 等于 班级）。
     - **顺序排列**：将建表、应用配置、业务系统创建等操作点按逻辑先后顺序排列。
     - **结果交付**：输出完整的、可直接操作的配置指南，流程结束。

## 注意事项
- **步骤稳定性**：第一步和第二步可能反复迭代，必须保证当前步骤稳定，严禁随意跳步。
- **持续磨合**：保持耐心，这是一个与用户需求反复磨合的过程，三步可能需要循环多次才能完全走通。
- **技能协同**：在任一阶段信息不足时，应主动调用检索或反问技能。
