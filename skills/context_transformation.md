# Skill: Context 转化与自动化索引 (Context Transformation & Auto-Indexing)

## 描述
该技能用于将用户上传的原始文件（如需求文档、技术说明、图片等）转化为结构化、层级分明的 Markdown Context，并自动维护 `context/index.md` 索引。它具备图像识别、链接标记、大文档切分以及逻辑自我进化的能力。

## 输入参数
- `file_path` (string): 待处理的原始文件路径。
- `target_directory` (string, optional): 指定存放转化后文件的目录（若不指定，系统将根据内容自动建议）。

## 执行逻辑

### 1. 文件解析与拆分
- **大文档处理**：若文档超过 5000 字或结构过于复杂，自动拆分为多个子模块（如：`Part 1: 背景`, `Part 2: 核心逻辑`）。
- **进度汇报**：在处理过程中实时向用户反馈进度（例如：“正在解析第 2/5 部分...”）。

### 2. 特殊内容处理
- **图片处理 (Images)**：
  - 识别 `.png`, `.jpg`, `.jpeg`, `.gif` 等格式。
  - **打标签**：根据图片在文档中的位置和上下文（前后的文字描述），为图片生成描述性标签（如：`#UI截图`, `#流程图`, `#架构设计`）。
  - **存储引用**：在生成的 Markdown 中保留图片链接，格式为 `![标签](path/to/image)`。
- **链接处理 (Links)**：
  - 识别文档中的外部链接或内部引用。
  - **标记**：根据上下文标记链接的作用（如：`[参考文档]`, `[API接口]`）。
- **未知内容处理**：
  - 若遇到无法识别或不知道如何转化的内容，**必须暂停并询问用户**。
  - **自我进化**：在用户给出处理建议后，将该规则更新到本 Skill 的“自定义处理规则”部分。

### 3. 层级化 Context 生成
- 将解析后的内容转化为规范的 Markdown 格式。
- 确保标题层级（H1, H2, H3）清晰，符合项目既有的 Context 风格。

### 4. 自动化索引维护
- **位置建议**：分析内容，匹配 `context/index.md` 中的既有分支。
- **层级调整**：若现有分支不匹配，**必须征求用户同意**（例如：“当前内容属于新领域，是否建议增加 `context/new_category/` 分支？”）。
- **更新索引**：在用户确认位置后，向 `context/index.md` 添加简略且有效的索引条目。

## 使用示例
- 用户上传：`new_feature.docx`（包含大量截图和外部链接）。
- Skill 动作：
  1. 提示：“正在将文档拆分为 3 部分处理...”
  2. 识别图片并根据上下文标注为 `![功能演示截图](...)`。
  3. 识别链接并标注为 `[后端 API 规范](...)`。
  4. 询问：“当前内容适合放入 `Logic` 还是新建 `Features` 分支？”
  5. 得到确认后，生成文件并更新 `index.md`。
