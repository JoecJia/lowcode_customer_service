# Skill: 信息补充与反问 (Clarifying Questions)

## 描述
当用户提供的信息不足以支持其他 Skill（如：场景方案建议、产品功能案例等）提供精准方案或答案时，调用此技能通过反问用户的方式获取关键缺失信息。

## 输入参数
- `current_context` (string): 当前已知的对话上下文或用户需求摘要。
- `missing_elements` (array): 已经识别出的、执行目标 Skill 所必须但当前缺失的信息项（如：用户量级、具体业务流程细节、预算范围等）。

## 执行逻辑
1. **上下文总结**：简要总结前文已确认的需求或信息点，让用户感知到系统已理解的内容。
2. **缺口识别**：根据目标任务的要求，明确指出当前还需要哪些具体信息才能继续。
3. **精准反问**：将缺失点转化为 1-3 个具体的、易于回答的问题返回给用户。
4. **结果传递**：将用户补充后的信息整合，返回给发起调用的原始 Skill。

## 使用示例
- 用户说：“我想搭一个审批流。”（信息过少）
- Skill 动作：总结“您需要搭建审批系统”，反问“审批的层级有哪些？”、“是否需要对接外部数据源？”等。
- 用户补充后：将信息传递给 `build_business_system` 技能继续执行。
